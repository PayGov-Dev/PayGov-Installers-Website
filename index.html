v1.0.8 PayGovPosInstall-1.0.8.msi
  • Add Version Checking
v1.0.7 PayGovPosInstall-1.0.7.msi
  • Updated to latest SDK. The app on the device must be version 1.11.11.18, version 1.11.12 will not work.
  • Now if the transaction fails, it will automatically retry. Then if it still fails and the customers is still processing the payment, it will prompt them to try again.
v1.0.6 PayGovPosInstall-1.0.6.msi
  • Make sure all communication to terminal logs in first, incase the terminal was unplugged between transactions
  • Made bug fixes recommended by AI
v1.0.5 PayGovPosInstall-1.0.5.msi
  • Create temporary orders if terminal was a success but payment couldn't be added to PayGov
  • User can try to enter order on failed payments
  • Ability to upload log file
  • Add signature to confirmation print page
  • Moved notification that the app is connected to terminal from a popup to the title window, it will only show up for 10 seconds after openning
v1.0.4 PayGovPosInstall-1.0.4.msi
  • Implemented retry logic for API calls to improve reliability
  • Added automatic relogin to EMV device during payment processing
v1.0.3 PayGovPosInstall-1.0.3.msi
  • Make First and Last name not required
v1.0.2 PayGovPosInstall-1.0.2.msi
  • Updated the MerchantE SDK dll. It increases the timeout to 2 minutes when talking to the terminal
v1.0.1.1 PayGovPosInstall-1.0.1.1.msi
  • Update terminal timeout to 2 minutes
v1.0.1 PayGovPosInstall-1.0.1.msi
